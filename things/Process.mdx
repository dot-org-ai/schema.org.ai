# Process

A business process representing a structured sequence of activities that accomplish a specific organizational objective. Processes can range from high-level strategic processes to detailed operational procedures and form the foundation of process.org.ai ontology.

## Properties

| Property | Type | Description | Inherited From |
|----------|------|-------------|----------------|
| `$type` | Text | Always "Process" | - |
| `$id` | URL | Unique identifier for the process | Thing |
| `name` | Text | The name of the process | Thing |
| `description` | Text | Description of what the process accomplishes | Thing |
| `identifier` | Text or PropertyValue | Process identifier (e.g., APQC code) | Thing |
| `url` | URL | URL to process documentation | Thing |
| `digital` | Number (0.0-1.0) | Digital maturity score | Thing |
| `category` | Text | Process category (e.g., "Develop Vision and Strategy") | Process |
| `processGroup` | Process | Parent process group | Process |
| `subprocess` | Process[] | Child processes or activities | Process |
| `task` | Task[] | Specific tasks within the process | Process |
| `input` | Thing[] | Required inputs | Process |
| `output` | Thing[] | Expected outputs | Process |
| `owner` | Person or Organization | Process owner | Process |
| `performer` | Occupation or Agent | Who performs the process | Process |
| `tool` | Tool[] | Tools used in the process | Process |
| `duration` | Duration | Typical process duration | Process |
| `frequency` | Text | How often the process runs | Process |
| `level` | Number | Process hierarchy level (1-5) | Process |
| `apqcCode` | Text | APQC PCF code if applicable | Process |
| `creator` | Person or Organization | The entity that defined the process | Thing |

## Types

The following types extend Process:

- **OperatingProcess**: Core operating processes
- **ManagementProcess**: Management and support processes
- **StrategicProcess**: Strategic planning processes
- **BusinessProcess**: End-to-end business processes
- **WorkflowProcess**: Automated workflow processes

  identifier: '1.0',
  apqcCode: '10.0.0',

  level: 1,

    {
      $type: 'Process',
      $id: 'https://process.org.ai/define.Business.Concept.and.Long-term.Vision',
      name: 'Define business concept and long-term vision',
      apqcCode: '10.1.0',
      level: 2
    },
    {
      $type: 'Process',
      $id: 'https://process.org.ai/develop.Business.Strategy',
      name: 'Develop business strategy',
      apqcCode: '10.2.0',
      level: 2

    {
      $type: 'Process',
      $id: 'https://process.org.ai/manage.Strategic.Initiatives',
      name: 'Manage strategic initiatives',

    $type: 'Occupation',

## Usage in Business-as-Code

```typescript
import { $ } from 'sdk.do'
import type { Process, Task, Agent } from 'schema.org.ai'

// Create a process
const process: Process = {
  $type: 'Process',
  $id: 'https://myorg.do/processes/my-process',
  name: 'My Business Process',
  description: 'A process that creates value',
  digital: 0.7
}

await $.Process.create(process)

// Query processes by category
const strategicProcesses = await $.Process.find({
  category: 'Strategic Planning'
})

// Find highly digital processes
const digitalProcesses = await $.Process.find({
  digital: { $gte: 0.8 }
})

// Get process hierarchy
const processWithChildren = await $.Process.get('process-id', {
  populate: ['subprocess', 'task']
})
```
