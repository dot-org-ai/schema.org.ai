# FeatureSection

A section highlighting product or service features with icons, images, titles, and descriptions. Supports grid, list, and alternating layouts.

## Type Hierarchy

[Thing](Thing.mdx) > [Intangible](Intangible.mdx) > [WebPageElement](WebPageElement.mdx) > **FeatureSection**

## Properties

### Core Properties

- **title** (Text): Section title
- **subtitle** (Text): Optional section subtitle
- **items** (Array): Array of feature items with title, description, icon/image
- **layout** (Text): Layout style (grid, list, alternating, cards, etc.)
- **columns** (Number): Number of columns for grid layout

### Feature Item Properties

Each item in the `items` array can include:
- **title** (Text): Feature title
- **description** (Text): Feature description
- **icon** (Text): Icon identifier (e.g., 'users', 'chart', 'shield')
- **image** (URL): Feature image or screenshot
- **link** (URL): Optional link to learn more
- **badge** (Text): Optional badge or label

## Usage in Business-as-Code

```typescript
import { $ } from 'sdk.do'
import type { FeatureSection } from 'schema.org.ai'

// Create a FeatureSection
const item: FeatureSection = {
  $type: 'FeatureSection',
  title: 'Everything You Need',
  layout: 'grid',
  columns: 3,
  items: [
    {
      title: 'Fast Performance',
      description: 'Lightning-fast load times',
      icon: 'zap'
    },
    {
      title: 'Secure by Default',
      description: 'Enterprise-grade security',
      icon: 'shield'
    },
    {
      title: 'Easy to Use',
      description: 'Intuitive interface',
      icon: 'smile'
    }
  ]
}

// Use semantic patterns
await $.FeatureSection.create(item)
const result = await $.FeatureSection.get('item-id')
```

## Layout Options

### Grid Layout
- Multiple columns (2, 3, or 4)
- Equal-height cards
- Icon or image per feature
- Best for: Multiple features with similar weight

### Alternating Layout
- Large image alternating left/right
- Detailed descriptions
- Specs and technical details
- Best for: Product features, in-depth explanations

### Cards Layout
- Elevated card design
- Rich content per card
- Call-to-action per card
- Best for: Services, offerings, tiered features

### List Layout
- Vertical list with icons
- Simple and scannable
- Compact presentation
- Best for: Benefits, checklist items
