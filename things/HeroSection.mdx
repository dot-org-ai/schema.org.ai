# HeroSection

A prominent hero section with headline, subtitle, CTA buttons, and hero image or video. Typically the first section visitors see on a landing page.

## Type Hierarchy

[Thing](Thing.mdx) > [Intangible](Intangible.mdx) > [WebPageElement](WebPageElement.mdx) > **HeroSection**

## Properties

### Core Properties

- **title** (Text): Main headline text
- **subtitle** (Text): Supporting subtitle or description
- **cta** (Action): Primary call-to-action button
- **secondaryCta** (Action): Optional secondary CTA
- **image** (URL): Hero image URL
- **backgroundVideo** (URL): Optional background video URL
- **badge** (Text): Small badge or label (e.g., "New", "Limited Time")
- **layout** (Text): Layout style (centered, split, fullscreen, etc.)

## Usage in Business-as-Code

```typescript
import { $ } from 'sdk.do'
import type { HeroSection } from 'schema.org.ai'

// Create a HeroSection
const item: HeroSection = {
  $type: 'HeroSection',
  title: 'Build Better Products Faster',
  subtitle: 'The all-in-one platform for modern product teams',
  cta: { text: 'Start Free Trial', url: '/signup' },
  image: '/hero-product.jpg'
}

// Use semantic patterns
await $.HeroSection.create(item)
const result = await $.HeroSection.get('item-id')
```

## Layout Variations

### Centered Layout
- Title and subtitle centered
- CTA buttons centered below
- Optional hero image below or as background
- Best for: Simple, focused messaging

### Split Layout
- Content on left, image on right (or reversed)
- Balanced visual hierarchy
- Best for: SaaS products, B2B services

### Fullscreen Layout
- Full viewport height
- Background video or image
- Overlay text and CTAs
- Best for: E-commerce, visual products

### Minimal Layout
- Clean, text-focused design
- Subtle hero image or none
- Emphasis on typography
- Best for: Portfolios, personal brands
